# CLAUDE.md — React Router Starter

Context for AI coding agents (and humans) working in this repo. This is a **reusable starter
template**, not a finished product: it ships one neutral example feature ("Items") that
exercises every layer. When you build a real feature, copy the Items pattern by analogy and
keep to the conventions below.

Detailed rules live in [`STYLE_GUIDE.md`](./STYLE_GUIDE.md) and
[`TESTING_GUIDE.md`](./TESTING_GUIDE.md); architecture in [`docs/ARCHITECTURE.md`](./docs/ARCHITECTURE.md);
how to make it yours in [`docs/CUSTOMIZATION.md`](./docs/CUSTOMIZATION.md). What was deliberately
dropped from the source app is recorded in [`NOT_CARRIED_FORWARD.md`](./NOT_CARRIED_FORWARD.md).

## Stack

- **React Router v7** (framework mode, SSR on by default — `react-router.config.ts` sets `ssr: true`).
- **React 19** + **TypeScript 5.8** (strict).
- **Tailwind CSS v4** — design tokens in the `@theme` block of `app/app.css`; no `tailwind.config.js`.
- **Vite 6** (plugins: `@tailwindcss/vite`, `@react-router/dev/vite`, `vite-tsconfig-paths`).
- **Jest 30** + ts-jest + Testing Library; **Playwright** for e2e.
- **ESLint 9** flat config + Prettier + Husky + lint-staged.
- **npm** only. Node 20 LTS (`.nvmrc`).

## Conventions

These are the rules every change is held to. The "why" for each is in `STYLE_GUIDE.md` /
`TESTING_GUIDE.md`.

### Data

- **SSR is on. Load data in route `loader`s and mutate in route `action`s** through the single
  typed `app/data/api.ts` client. **Never call `fetch()` directly in a component.** (For
  live/polling screens, the `useEffect`+`useState` alternative is documented in
  `docs/CUSTOMIZATION.md` — it is the exception, not the default.)
- **Drive list state (page, search, sort) from the URL** via `useSearchParams` + `useOrderParams`;
  the loader reads the search params. Never hold list/filter state in component `useState`.
- **All HTTP goes through `apiRequest<T>`**, which throws `ApiError(status, message)` on failure.
  Loaders let `ApiError` bubble to the `ErrorBoundary`; actions catch it and return field errors
  to the form.

### Routing

- **Use generated route types**: `import type { Route } from './+types/<route>'` and type
  loaders/actions/components with `Route.LoaderArgs` / `Route.ActionArgs` / `Route.ComponentProps`.
  Never hand-type params. (Generated by `react-router typegen`; run `npm run typecheck` after route changes.)
- **Every navigable resource follows the file convention**: `<resource>/<resource>.tsx` (list),
  `<resource>-layout.tsx` (parent that loads shared data and exposes it via `Outlet` context),
  `<resource>-detail.tsx`, `<resource>-new.tsx`, `<resource>-edit.tsx`. Each route exports
  `meta()` returning `buildMetaData(...)`.

### Components & styling

- **Style with variant maps** (`Record<Variant, string>`) composed via the single `cn()` helper
  (`~/helpers/cn`). No inline styles, no `style` props, no arbitrary Tailwind values, no ad-hoc
  string concatenation.
- **Tailwind v4 tokens live in the `@theme` block of `app/app.css`** (colors `primary`/`secondary`/
  `neutral`/`success`/`warning`/`error`, spacing, and a text scale `text-display`/`text-h1..h3`/
  `text-body`/`text-small`). Reference them as utilities. There is no `tailwind.config.js`.
- **Use the `Heading` component** (`level` for visual size, `as` for the semantic tag) instead of
  raw `<h1>`–`<h6>`.
- **Use `ui/` primitives before writing new ones.** Import from the barrel: `~/components/ui`.
- **Icons** come from `~/components/icons/Icon` (the `makeIcon` factory) — never mix raw
  `lucide-react` imports with the wrapped ones.
- **Every interactive element** has an ARIA role/label and keyboard support.

### State

- **Compose cross-cutting state in `AppProviders`; one concern per provider.** Expose a `use<Name>()`
  hook that **throws when used outside its provider**. Memoize context values (`useMemo`) and
  handlers (`useCallback`).

### Tooling

- **Access `import.meta.env` only in `app/helpers/env.ts`**; access `localStorage` only via the
  `safe*` wrappers in `app/helpers/storage.ts`. Optional integrations (Sentry/Logfire) self-disable
  when their env var is unset.
- **Path alias `~/*` → `app/*`** for all cross-folder imports (kept in sync across `tsconfig`,
  `vite-tsconfig-paths`, and the jest `moduleNameMapper`).

### Testing

- **One reason per test.** Query priority: `getByRole` > `getByLabelText` > `getByPlaceholderText`
  > `getByText` > `getByTestId`. No snapshots. No comments. Build data with typed factories.
- **Render via the shared helpers** (`renderWithRouter` / `renderWithProviders`); **mock the api
  layer, not internal logic.** Route/loader/action tests use `createRouteStub` with a mocked api.
- **Coverage thresholds** (CI gates): statements 80 / branches 75 / functions 70 / lines 75. The
  framework shell (`root.tsx`, `entry.client.tsx`, `routes.ts`, `**/+types/**`) is excluded.
- **E2E** runs sequentially (`workers: 1`) with a one-time `auth.setup.ts` storage-state fixture
  reused across specs.

## What to customize first

This is a template — these are the things to change before it becomes "your app":

1. **`app/types/`** — replace `items.ts` with your resource types; re-export from `index.ts`.
   Trim `core.ts` (`User`, `ListApiResponse<T>`) to what you actually need.
2. **`app/data/api.ts`** — point `apiBaseUrl` at your backend, and replace `itemsApi` with your
   resource objects (mirror the `list`/`get`/`create`/`update`/`remove` shape).
3. **`app/routes.ts`** + `app/routes/` — register your resource tree following the file convention.
4. **`app/app.css` `@theme` tokens** — set your color palette, text scale, and spacing. This is
   the one place to reskin.
5. **`APP_NAME`** in `app/helpers/meta.ts` — currently `'RR Starter'`; this is the page-title
   suffix and is meant to be changed.

Then delete the example feature (`app/routes/items/`, `app/components/items/`, `app/types/items.ts`
and its tests/mocks) once your own resource is in place.

See [`docs/CUSTOMIZATION.md`](./docs/CUSTOMIZATION.md) for the full recipe (alias, providers,
tokens, disabling SSR, env vars, monitoring, analytics, the component-fetch alternative, and the
SSR token-auth caveat).
